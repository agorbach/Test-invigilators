# תרגיל 3 — System Design עם AI

## מטרת התרגיל

לתכנן מערכת בצורה מקצועית לפני כתיבת קוד.

בתרגיל זה תעבדו על:
- חשיבה מערכתית
- תכנון Database
- בניית Architecture
- עבודה נכונה עם AI
- פירוק מערכת מורכבת לחלקים

---

# רקע

בתרגיל הקודם בניתם מערכת לניהול משגיחים באמצעות AI.

כעת נעבור לשלב הבא:

לא רק לייצר קוד —
אלא לתכנן מערכת כמו Software Engineers אמיתיים.

---

# למה זה חשוב?

AI יכול לייצר קוד.

אבל:
- AI לא מבין Business Logic בצורה מושלמת
- AI לא מחליף Architecture
- AI לא מחליף System Design

מפתחים מקצועיים צריכים לדעת:
- לתכנן Database
- להבין Relationships
- לבנות Flows
- לחשוב בצורה מערכתית

---

# מטרות הלמידה

בסיום התרגיל תדעו:
- לבנות ERD
- להגדיר Relationships
- לחשוב על System Flow
- לכתוב User Stories
- להשתמש ב-AI לצורך Design ולא רק Code Generation

---

# תיאור המערכת

עליכם לתכנן מערכת לניהול משגיחים במכללה.

המערכת כוללת:
- מבחנים
- משגיחים
- שיבוצים
- אילוצים
- זמינויות
- דוחות
- Dashboard ניהולי

---

# חלק 1 — ERD

## מה זה ERD?

ERD = Entity Relationship Diagram

תרשים שמציג:
- טבלאות
- קשרים
- מבנה Database

---

# למה ERD חשוב?

Database טוב =
- מערכת יציבה יותר
- פחות באגים
- פיתוח קל יותר
- Scalability טובה יותר

---

# מה צריך לכלול ב-ERD

עליכם ליצור ERD מלא עבור המערכת.

ה-ERD חייב לכלול לפחות:

## Tables

### Exams
- id
- course_name
- exam_date
- start_time
- end_time
- room_id
- students_count
- moed

---

### Invigilators
- id
- first_name
- last_name
- phone
- email
- can_read
- max_daily_hours
- active

---

### Assignments
- id
- exam_id
- invigilator_id
- assigned_at
- status

---

### Rooms
- id
- building
- room_number
- quiet_room
- computer_room

---

### Availability
- id
- invigilator_id
- available_date
- available_from
- available_to

---

### Constraints
- id
- invigilator_id
- constraint_type
- notes

---

# Relationships

הגדירו Relationships בין הטבלאות.

דוגמאות:

```text
Exam → Many Assignments

Invigilator → Many Assignments

Room → Many Exams

Invigilator → Many Availability Records
```

# חלק 2 — System Flow

## מה זה System Flow?

הדרך שבה משתמש מתקדם במערכת.

---

# מה צריך להכין

צרו Flow בסיסי של המערכת.

---

# Example Flow

```text
Upload Exams CSV
↓
Validate Data
↓
Import Database
↓
Run Assignment Engine
↓
Detect Conflicts
↓
Assign Invigilators
↓
Generate Reports
↓
Send Emails
```

---

# דרישות

ה-Flow צריך לכלול:
- Import

- Assignment
- Dashboard
- Reports
- Notifications

# חלק 3 — User Stories

## מה זה User Story?

דרך לתאר Feature מנקודת מבט של משתמש.

---

# מבנה קלאסי

```text
As a [user],
I want [action],
So that [benefit]
```

---

# דוגמאות

```text
As a coordinator,
I want to assign invigilators automatically,
So that I can save time.
```

```text
As an invigilator,
I want to receive email notifications,
So that I know my upcoming assignments.
```

```text
As a manager,
I want to see scheduling conflicts,
So that I can fix issues quickly.
```

---

# דרישות

כתבו לפחות:
- 5 User Stories
